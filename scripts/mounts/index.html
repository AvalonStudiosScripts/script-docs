    avalon_mounts

    Script per l'utilizzo di creature cavalcabili in FiveM. Permette ai giocatori di evocare, montare e gestire creature fantastiche (draghi, grifoni, pegasi, cavalli, alpaca) tramite comandi o item nell'inventario.

    Versione: 1.0 | Autore: Mino

    Dipendenze

    Resource Obbligatoria Note
    ox_lib ✅ Notifiche e utility
    ox_inventory ❌ Solo se CONFIG.items = true
    ox_fuel ❌ Mantiene carburante al massimo automaticamente
    qbx_core ❌ Integrazione chiavi QBox
    qbx_vehiclekeys ❌ Integrazione chiavi QBox

    Installazione

    ensure avalon_mounts

    Configurazione

    Impostazioni Generali

    Opzione Tipo Default Descrizione
    CONFIG.command boolean true Abilita i comandi per evocare le creature
    CONFIG.commandName string "mount" Nome del comando evoca (/mount)
    CONFIG.delCommand string "delMount" Nome del comando congeda (/delMount)
    CONFIG.items boolean true Usa item da inventario per evocare (richiede ox_inventory)
    CONFIG.invincible boolean false Rende la creatura invincibile
    CONFIG.maxrangedespawn number 200.0 Distanza (unità) dopo la quale la creatura si despawna automaticamente
    CONFIG.timeForSpawnAfterDeaths number 10000 Tempo di attesa (ms) per evocare dopo la morte

    Tasti

    Opzione Tasto default Descrizione
    CONFIG.keyMount 51 (E) Monta / Smonta
    CONFIG.keyDelete 74 (H) Congeda la creatura
    CONFIG.keyFlymode 73 (X) Attiva/disattiva modalità volo

    Sistema Volo

    CONFIG.flyWithTime  = false  -- false = volo illimitato, true = volo a tempo
CONFIG.flyTimeMax   = 60000  -- ms di volo disponibili (se flyWithTime = true)
CONFIG.flyCooldown  = 60000  -- ms di ricarica prima di poter volare di nuovo

    Velocità Massima

    Valore di moltiplicazione interno (non km/h reali, da testare sul server).

    CONFIG.MaxSpeed = {
    ["necrodragon"] = 100.0,
    ["griffon"]     = 50.0,
    ["pegaso"]      = 70.0,
    ["alpaca"]      = 50.0,
    ["horse"]       = 70.0,
}

    API — Client-Side

    Tipo: Client-Side

    Restituisce l'oggetto API con tutte le funzioni pubbliche dello script.

    local MountsAPI = exports['avalon_mounts']:API()

    Funzioni dell'oggetto API

    Funzione Descrizione
    MountsAPI.getData(model) Ottiene i dati di configurazione di una creatura specifica
    MountsAPI.getAllData() Ottiene i dati di tutte le creature
    MountsAPI.getBlacklist(vehicle) Controlla se un veicolo è in blacklist
    MountsAPI.spawn(name, color) Evoca una creatura (name = tipo, color = numero variante)
    MountsAPI.despawn() Congeda la creatura attiva
    MountsAPI.isSpawned() Restituisce true se una creatura è evocata
    MountsAPI.isOnMount() Restituisce true se il giocatore è montato
    MountsAPI.getVehicle() Restituisce l'entità veicolo della creatura
    MountsAPI.sendInfo(data) Invia dati di stato della creatura
    MountsAPI.deleteEnt(ent) Elimina un'entità specifica
    MountsAPI.playAnimation(prop, anim, dict, loop) Riproduce un'animazione sulla creatura
    MountsAPI.syncInvisible(ent, bool) Sincronizza visibilità tra tutti i client
    MountsAPI.syncHair(ent, head) Sincronizza variante capelli/testa

    Esempi

    -- client/main.lua del tuo script
local MountsAPI = exports['avalon_mounts']:API()

-- Evoca un necrodragon (variante 1)
MountsAPI.spawn("necrodragon", 1)

-- Controlla se il giocatore ha una creatura evocata
if MountsAPI.isSpawned() then
    print("Il giocatore ha una creatura attiva")
end

-- Ottieni l'entità del mount attivo
local mountVehicle = MountsAPI.getVehicle()
if mountVehicle then
    local coords = GetEntityCoords(mountVehicle)
    print("Mount alle coordinate: " .. tostring(coords))
end

-- Congeda la creatura
MountsAPI.despawn()

    Export Item Dinamici — Client-Side

    Tipo: Client-Side

    Per ogni variante creatura viene registrato automaticamente un export con il nome dell'item. Richiede CONFIG.items = true.

    Il nome degli export segue il formato: nomeCreatura_numeroVariante

    -- Evoca necrodragon variante 1 tramite item
exports['avalon_mounts']:necrodragon_1()

-- Evoca necrodragon variante 2
exports['avalon_mounts']:necrodragon_2()

-- Evoca griffon variante 1
exports['avalon_mounts']:griffon_1()

-- Evoca pegaso variante 2
exports['avalon_mounts']:pegaso_2()

-- Evoca horse variante 1
exports['avalon_mounts']:horse_1()

-- Evoca alpaca variante 3
exports['avalon_mounts']:alpaca_3()

    Uso con ox_inventory

    Configura ogni item mount in ox_inventory con la callback che chiama l'export corrispondente. Lo script lo gestisce automaticamente quando CONFIG.items = true.

    Esempio Completo

    -- client/main.lua di un altro script

local MountsAPI = exports['avalon_mounts']:API()

-- Menu per scegliere la creatura
RegisterCommand('menu_mounts', function()
    lib.registerContext({
        id    = 'menu_mounts',
        title = 'Scegli la tua creatura',
        options = {
            {
                title    = 'Necrodragon',
                description = 'Drago del male — Volo illimitato',
                onSelect = function()
                    MountsAPI.spawn('necrodragon', 1)
                end
            },
            {
                title    = 'Pegaso',
                description = 'Cavallo alato — Velocità alta',
                onSelect = function()
                    MountsAPI.spawn('pegaso', 1)
                end
            },
            {
                title    = 'Horse',
                description = 'Cavallo classico',
                onSelect = function()
                    MountsAPI.spawn('horse', 1)
                end
            },
        }
    })
    lib.showContext('menu_mounts')
end)

-- Congeda da comando
RegisterCommand('congeda', function()
    if MountsAPI.isSpawned() then
        MountsAPI.despawn()
    end
end)

    Note Integrazione ox_fuel

    Se ox_fuel è presente, il carburante della creatura viene mantenuto automaticamente al massimo ogni 5 secondi. Non è richiesta nessuna configurazione aggiuntiva.

    Note Integrazione QBox

    Se qbx_core e qbx_vehiclekeys sono presenti, le chiavi del veicolo creatura vengono assegnate automaticamente al giocatore ogni 10 secondi finché la creatura è evocata.
